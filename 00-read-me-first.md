# 做出一个能跑的 Mini Codex

[English](./00-read-me-first.en.md)

如果你已经会写业务代码，也已经在用 Codex、Cursor 或 Claude Code，但还说不清“智能体到底怎么完成一个工程任务”，这套课就是为你准备的。

你不会只多学几个 prompt。你会从一个空目录开始，做出一个本地 AI 编程助手，并掌握一套可以迁移到其它场景的智能体编程能力：

```text
输入一句代码修改需求
-> 保存成一条可继续处理的任务
-> 读取示例项目里的文件
-> 给出修改计划
-> 等你确认
-> 修改文件
-> 运行检查
-> 保存整个过程
-> 做成一个桌面界面
```

## 1. 学完以后你会得到什么

### 1.1 一个能展示、能讲清、能继续升级的项目

完成基础课后，你会做出一个最小但完整的 Mini Codex：

```text
输入一句代码修改需求
-> 保存任务
-> 读取项目文件
-> 生成计划
-> 等你确认
-> 生成改动草稿
-> 修改目标文件
-> 运行检查
-> 保存过程记录
-> 复盘效果
-> 在桌面界面里观察整个过程
```

它不是官方 Codex 的完整替代品，但它是一个很适合作为作品集、面试讲解和二次开发起点的项目。你能展示的不只是“我调用了大模型”，而是：

- 我知道一个 Coding Agent 如何保存任务
- 我知道它为什么要先计划、再等待确认
- 我知道模型为什么不能直接随意改文件
- 我知道 diff、verify、trace、eval 分别解决什么工程风险
- 我知道如何把命令行能力包装成 API 和桌面界面

### 1.2 一套可迁移的智能体编程能力

Mini Codex 只是训练载体。真正值得带走的是这些能力：

- 把一句需求变成可跟踪的任务
- 让 AI 先给计划，确认后再改代码
- 控制哪些动作必须经过你同意
- 修改前先看改动草稿
- 修改后运行检查
- 保存过程，方便回看和复盘
- 把命令行能力做成桌面界面

学完后，你可以把这套结构迁移到更多工具里：

```text
代码审查助手：先读改动，再生成风险清单和验证建议
文档生成助手：先收集资料，再生成大纲、正文和审阅记录
数据检查助手：先读取文件，再给出清洗计划和异常报告
运维排查助手：先收集日志，再给出排查步骤和执行证据
业务流程助手：先拆任务，再按权限执行、记录和复盘
```

### 1.3 求职、转型和自用工具的工程证据

这门课不能承诺你学完就一定拿到 offer，也不会把“AI 应用工程师”简化成几个术语。但它能帮你补上很多岗位真正会追问的能力证据：

- 能不能把 AI 工具做成可控的工程系统？
- 能不能解释 Agent Run、工具注册、权限审批、验证和回放？
- 能不能说明 prompt 和 runtime policy 的区别？
- 能不能让 AI 输出变成可审查、可恢复、可评测的过程？
- 能不能基于一个最小系统继续做真实模型、RAG、MCP、Memory 或工作台？

如果你想找 AI 应用工程师、Agent 工程师、AI 提效工具开发、内部工具平台这类方向，这个项目可以成为你的讲解材料。如果你暂时不换工作，也可以把它改造成自己的研发助手、团队小工具或某个垂直场景的智能体工作台。

## 3. 每个 Lab 怎么学

每一节都按这个节奏走：

```text
先看这一节要做出什么
-> 跟着创建文件和写代码
-> 运行命令看结果
-> 对照验收清单
-> 再回头理解这一节训练的设计能力
```

你可以在 `design-journal.md` 里记录 3 个问题：

1. 这一节解决了什么工程问题？
2. 如果不这样设计，后面会在哪里出错？
3. 如果我做进阶版，第一处要增强哪里？

记录时先用自己的话写清楚问题、结果和风险；遇到不确定的工程名，再回到对应 Lab 的代码和运行结果里核对。

## 4. 先别急着做的事

- 先别还没做完基础闭环，就去做外部工具连接、长期记忆或自动化扩展。
- 先别看到官方工具有某个功能，就马上塞进基础课。
- 先别把演示跑通当成真的掌握。
- 先别跳过验收清单。

基础课的目标是把最小闭环跑稳。进阶课再一步步补更完整的改动解析、撤销能力、项目资料检索、长期记忆、外部工具连接、代码版本管理和复盘面板。
