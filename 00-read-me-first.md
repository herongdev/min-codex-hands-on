# 做出一个能跑的 Mini Codex

你会从一个空目录开始，做出一个本地 AI 编程助手：

```text
输入一句代码修改需求
-> 保存成一条可继续处理的任务
-> 读取示例项目里的文件
-> 给出修改计划
-> 等你确认
-> 修改文件
-> 运行检查
-> 保存整个过程
-> 做成一个桌面界面
```

## 1. 做完以后你能得到什么

完成 v2 基础课程后，你应该能做出一个最小但完整的 Mini Codex：

```text
输入一句代码修改需求
-> 保存任务
-> 读取项目文件
-> 生成计划
-> 等你确认
-> 生成改动草稿
-> 修改目标文件
-> 运行检查
-> 保存过程记录
-> 复盘效果
-> 在桌面界面里观察整个过程
```

它不是官方 Codex 的完整替代品，但能让你逐步学会这些能力：

```text
把一句需求变成可跟踪的任务
让 AI 先给计划，确认后再改代码
控制哪些动作必须经过你同意
修改前先看改动草稿
修改后运行检查
保存过程，方便回看和复盘
把命令行能力做成桌面界面
```

## 3. 每个 Lab 怎么学

每一节都按这个节奏走：

```text
先看这一节要做出什么
-> 跟着创建文件和写代码
-> 运行命令看结果
-> 对照验收清单
-> 再回头理解这一节训练的设计能力
```

你可以在 `design-journal.md` 里记录 3 个问题：

```text
这一节解决了什么工程问题？
如果不这样设计，后面会在哪里出错？
如果我做进阶版，第一处要增强哪里？
```

记录时先用自己的话写清楚问题、结果和风险；遇到不确定的工程名，再回到对应 Lab 的代码和运行结果里核对。

## 4. 先别急着做的事

```text
先别还没做完基础闭环，就去做外部工具连接、长期记忆或自动化扩展。
先别看到官方工具有某个功能，就马上塞进基础课。
先别把演示跑通当成真的掌握。
先别跳过验收清单。
```

基础课的目标是把最小闭环跑稳。进阶课再一步步补更完整的改动解析、撤销能力、项目资料检索、长期记忆、外部工具连接、代码版本管理和复盘面板。
